# Vitality Bowls — AI Growth Engine Dashboard

A front-end dashboard built as part of a group hackathon project — one that we won. I owned the design and front-end development — concept, design system, and the full build.

**Live demo:** [drehandley.github.io/vitality-bowls](https://drehandley.github.io/vitality-bowls)

---

## What it is

A multi-section business dashboard concept built around the Vitality Bowls Stamford brand, created as part of a hackathon. I built out a complete design system from scratch — brand colors, typography, spacing, components, all of it — and designed the full UI on top of it.

Sections include an outreach pipeline, prospect scoring, a content & ad studio, campaign sequencing, and a product roadmap view.

---

## What I built

- **Custom design system** — CSS variables for color, spacing, radius, typography, and brand tokens (purple `#4B2150`, green `#6AB04C`)
- **Multi-section tabbed layout** — sticky nav, sidebar, and responsive main content area
- **Interactive components** — modals, filters, sliders with live estimates, toast notifications, scroll reveal animations
- **Editorial typography** — Newsreader serif, Geist sans, JetBrains Mono — mixed intentionally for a premium feel
- **Responsive design** — layout adapts cleanly at 980px and below
- **Micro-details** — paper grain texture overlay, ambient gradient drift animation, animated sequence steps, hover states throughout

cd vitality-bowls

## What I learned

Building a real design system from scratch taught me how much thought goes into making UI feel cohesive — every spacing value, color, and font choice compounds. Getting the small things right (hover states, transitions, type sizing) is what separates something that looks designed from something that just looks built.
